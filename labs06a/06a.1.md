# Lab 6a.1 – Namespace Preparation and Policy Configuration

## Learning Objectives

In this section you will:

- Create isolated namespaces.
- Apply a Pod Security Standard.
- Configure a ResourceQuota.
- Create and review NetworkPolicies.
- Deploy an NGINX Ingress Controller.

---

## Why are we doing this?

Modern Kubernetes clusters rarely allow workloads to run without restriction. Platform teams typically enforce security, resource governance and network segmentation before application teams are permitted to deploy workloads.

In this lab you will build this platform layer before deploying any applications.

---

## 1. Create the namespaces

```bash
kubectl create namespace webserver
kubectl create namespace ingress
kubectl label ns webserver app=webserver
kubectl label ns ingress app=nginx-ingress
```

Verify:

```bash
kubectl get ns --show-labels
```

## 2. Apply the Restricted Pod Security Standard

```bash
kubectl label ns webserver pod-security.kubernetes.io/enforce=restricted
```

The Restricted Pod Security Standard validates every new Pod admitted into the namespace. Pods attempting to run as root or without appropriate security settings will be rejected.

## 3. Create a ResourceQuota

```bash
kubectl create quota webserver-quota --hard=pods=5,cpu=2,memory=2G --dry-run=client -o yaml > ws-quota.yaml
kubectl apply -n webserver -f ws-quota.yaml
```

Verify:

```bash
kubectl describe resourcequota webserver-quota -n webserver
```

## 4. Prepare the NetworkPolicies

```bash
cp ~/qakf-3day/solutions/lab6a/netpol_webserver1.yaml netpol_webserver.yaml
cp ~/qakf-3day/solutions/lab6a/netpol_ingress.yaml netpol_ingress.yaml
```

Edit **netpol_webserver.yaml** so that ingress traffic is permitted from the **ingress** namespace on TCP port **8080**.

Solution:

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: webserver-netpol
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          app: nginx-ingress
    ports:
    - protocol: TCP
      port: 8080
```

Apply both policies:

```bash
kubectl apply -n webserver -f netpol_webserver.yaml
kubectl apply -n ingress -f netpol_ingress.yaml
```

Verify:

```bash
kubectl get networkpolicy -A
```

## 5. Deploy the NGINX Ingress Controller

```bash
helm -n ingress install nginx-ingress oci://ghcr.io/nginx/charts/nginx-ingress --version 2.5.2 --set controller.kind=daemonset
```

Verify:

```bash
kubectl get pods -n ingress
kubectl get svc -n ingress
```

## Summary

The cluster is now prepared with namespace isolation, pod security, resource governance, network policies and an ingress controller. In the next lab you will deploy and progressively harden an application until it satisfies all enforced policies.
